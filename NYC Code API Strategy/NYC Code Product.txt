  def github_to_deepwiki(github_url):
      pattern = r'github\.com/([^/]+)/([^/]+)'
      match = re.search(pattern, github_url)
      owner = match.group(1)
      repo = match.group(2).replace('.git', '')
      return f"https://deepwiki.com/{owner}/{repo}"

  DeepWiki Scraper Architecture (deepwiki_to_md/deepwiki_to_md.py):

  The scraper has three scraping strategies with priority order:

  Priority 1: DirectMarkdownScraper (Highest Priority)

  - Fetches markdown content directly from DeepWiki's API endpoints
  - Bypasses HTML parsing entirely for faster, cleaner extraction

  Priority 2: Alternative Scraper (scrape_deepwiki)

  - Uses HTTP requests to fetch raw HTML
  - Parses specific HTML structure from DeepWiki pages
  - Fallback when DirectMarkdownScraper fails

  Priority 3: Standard Scraper (Lowest Priority)

  - Fetches HTML via requests library
  - Uses BeautifulSoup for parsing
  - Extracts navigation items from ul.flex-1.flex-shrink-0.space-y-1.overflow-y-auto.py-1
  - For each navigation item:
    a. Fetches page HTML
    b. Extracts main content using CSS selectors (tries main article, main, .content, etc.)
    c. Converts HTML to Markdown using markdownify library
    d. Removes first 28 lines (header/navigation boilerplate)
    e. Fixes markdown links by converting [text](url) → [text]()
    f. Saves to .md files

  Content Extraction (deepwiki_to_md.py:265-336):
  # Tries multiple selectors in order of specificity:
  selectors = [
      'main article',           # Most specific
      'main .content',
      'main',
      'article',
      '.content',
      '.markdown-body',
      'div.container div.row'   # Most general
  ]

  User Request
      ↓
  [POST /api/generate-docs]
      ↓
  GitHub URL → DeepWiki URL conversion
      ↓
  Create Drive folder immediately
      ↓
  Return Drive link to user
      ↓

      ↓
  Share folder + Create sheet entry
      ↓
  Add to processing queue
      ↓
  [Queue Worker Thread]
      ↓
  Subprocess: deepwiki_to_md scraper
      ↓ (tries 3 strategies in order)
  ├─ DirectMarkdownScraper (API)
  ├─ Alternative Scraper (HTTP + BeautifulSoup)
  └─ Standard Scraper (Navigation extraction)
      ↓
  Saves .md files to output/{owner}_{repo}/{repo}/md/
      ↓
  Create Master Doc in Drive
      ↓
  Upload all .md files to Drive folder
      ↓
  Update Sheet status: "Complete"
      ↓
  Delete local files (cleanup)

  ---
  Key Design Decisions

  1. Immediate folder creation: Users get Drive link instantly, scraping happens asynchronously
  2. Sequential queue: Prevents overwhelming DeepWiki servers with parallel requests
  3. Three-tier scraping strategy: Robustness through fallback mechanisms
  4. Automatic cleanup: Prevents server disk from filling up
  5. Ticket system: Users can track their position in the queue
  6. Master document: Provides user instructions and file index in Drive

  ---
  Key Design Decisions

  1. Immediate folder creation: Users get Drive link instantly, scraping happens asynchronously
  2. Sequential queue: Prevents overwhelming DeepWiki servers with parallel requests
  3. Three-tier scraping strategy: Robustness through fallback mechanisms
  4. Automatic cleanup: Prevents server disk from filling up
  5. Ticket system: Users can track their position in the queue
  6. Master document: Provides user instructions and file index in Drive

  This architecture ensures reliability, user experience, and resource management while handling potentially
  long-running scraping operations.
