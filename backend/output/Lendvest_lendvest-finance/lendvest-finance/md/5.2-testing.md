Testing Best Practices
----------------------

When working with the LVLidoVault test suite, follow these guidelines:

### Mock and Simulation Patterns

* **Price Manipulation**: Use `stEthPerTokenMock()` for simulating market conditions
* **Time Progression**: Use `vm.warp()` to simulate epoch durations and claim periods
* **User Actions**: Create distinct addresses for different user types (lenders, borrowers, CLs)
* **Balance Management**: Use `deal()` to provide test accounts with appropriate token balances

### Test Isolation

Each test should:

* Start from the same initialized state (via `setUp()`)
* Clean up any state changes that might affect other tests
* Use unique user addresses to avoid interference
* Reset any mock values to default states

### Automation Testing

When testing Chainlink automation:

* Always check `checkUpkeep()` before calling `performUpkeep()`
* Decode the `performData` to understand which task will execute
* Use the `forwarder` address for automation calls
* Verify task completion by checking contract state changes
