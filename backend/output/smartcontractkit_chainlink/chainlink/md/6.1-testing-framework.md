
Purpose and Scope
-----------------

The Testing Framework provides comprehensive testing infrastructure for the Chainlink node implementation, covering unit testing, integration testing, end-to-end testing, performance testing, and chaos engineering. This document covers the test execution pipelines, test tooling, and infrastructure orchestration used to validate the Chainlink codebase across multiple environments and scenarios.

For information about the CI/CD build and deployment systems, see [CI/CD Pipeline](). For details about the build system and containerization, see [Build and Deployment System]().

Testing Infrastructure Overview
-------------------------------

The testing framework is built around multiple layers of test execution, from lightweight unit tests to comprehensive end-to-end scenarios running in Kubernetes environments.

**Testing Infrastructure Components**

| Component | Purpose | Entry Point |
| --- | --- | --- |
| `TestScripts` | CLI command testing with testscript framework | `main_test.go:TestScripts` |
| `go_core_tests` | Unit tests for core functionality | `tools/bin/go_core_tests` |
| `go_core_tests_integration` | Integration tests with external dependencies | `tools/bin/go_core_tests_integration` |
| `go_core_fuzz` | Fuzz testing for security and robustness | `tools/bin/go_core_fuzz` |
| `go_core_race_tests` | Race condition detection tests | `tools/bin/go_core_race_tests` |
| E2E Tests | End-to-end scenario testing | `.github/e2e-tests.yml` |
